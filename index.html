checking…

  • App shell cached
  • Currently online

Push notifications

        checking…
      

      
The browser asks for permission, generates a PushSubscription tied to a VAPID public key, and stores it locally. The "Send test push" button POSTs that subscription to /api/send-push, which uses the matching private key to push a notification back.

      
    
        
  • Push API supported

  • Notification permission
    • 
        
  • Subscription active
    • 
        
  • VAPID public key fetched

        Cross-device broadcast. Each device that subscribes is registered server-side in Vercel KV.
        The broadcast button below posts to /api/broadcast, which fans out to every stored subscription —
        so a push from desktop reaches the iPhone (and vice versa). Dead subscriptions get cleaned up on 410.
